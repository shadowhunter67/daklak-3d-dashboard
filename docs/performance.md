# Performance

`npm run build:metrics` publishes exact JavaScript/CSS/image totals, chunk counts, Three.js chunk size, geometry sizes and budget comparison to `reports/build-metrics.json` and `.md`.

Heavy WebGL code is split behind React lazy boundaries. Direct `?view=2d` startup avoids mounting and fetching it. `maplibre-gl` + `pmtiles` (the detail map, `?view=map`) are a second independently lazy-loaded boundary — opening the 3D overview or the 2D directory never fetches them, and opening the detail map never fetches `three-vendor`/`AdministrativeMap`. `StatPanel`'s small summary bar chart (used by the 3D/2D dashboard panels) is its own lazy chunk too, but is now plain SVG/CSS — no charting library is bundled anywhere in this repository (ECharts was removed; see the "Licensing + performance hardening" entry below). Terrain uses four precomputed textures and a 192×160 mesh (61,440 triangles), while polygon hit testing first filters by bounding boxes and is throttled to animation frames.

Executive Overview (Phase 2A, the default landing since every visitor loads it) is deliberately **not** a lazy boundary — it never imports Three.js or MapLibre (its status chart is plain CSS/SVG, not a charting library), so eagerly bundling it does not pull any heavy renderer into the initial load; direct `?view=3d`/`?view=2d`/`?view=map` startup still never fetches it needlessly since it's already part of the same eager chunk those entry points share. Landing on it added ~51KB raw / ~18KB gzip to the main chunk (`totalJavaScriptGzipBytes` 909,559 → 923,068 bytes; budget ceiling 950,000 — see `reports/performance-budget.json`); `check:budget` still passed but headroom was closer (~27KB gzip) than before that feature.

**Licensing + performance hardening (source-available transition):** `StatPanel`'s summary bar chart was rewritten from ECharts (core + `BarChart` + `GridComponent` + `SVGRenderer` + `echarts-for-react`, already modular-imported, not the full bundle) to a small hand-written inline SVG component — the chart itself is always exactly 3 bars, so a general-purpose charting library was never load-bearing here. `echarts`/`echarts-for-react` were removed from `package.json` entirely. This is the only chart in the codebase; there is no other ECharts usage left to migrate. `StatPanel`'s own lazy chunk dropped from ~503KB raw / ~172KB gzip to ~3KB raw / ~1.5KB gzip. `totalJavaScriptGzipBytes` dropped from 933,642 to 764,653 bytes against the unchanged 950,000-byte ceiling — headroom went from ~16KB to ~185KB gzip.

`npm run check:budget` fails on total JavaScript raw/gzip, largest JavaScript gzip, total texture bytes, largest asset, total build size, or a build containing no JavaScript/assets. Exact results live in `reports/performance-budget.json`; thresholds are regression ceilings, not targets. The limits were raised once (2026-07) specifically to admit the `maplibre-gl` vendor chunk (~271KB gzip, a normal size for that library) that the detail map's lazy boundary introduces — a user who never opens the detail map never downloads it; see `scripts/check_build_budget.mjs`'s comment and the "does not load the MapLibre chunk when starting in the 3D overview" / "loads the MapLibre chunk only once the detail map is opened" E2E tests for the actual guarantee that matters, which this aggregate ceiling only approximates.

Vitest performs 1,000 representative hit tests with a deliberately broad one-second CI ceiling. Real FPS, GPU memory, and thermal behavior depend on hardware and must be checked on representative physical devices before a high-traffic release.

`src/utils/graphicsQuality.ts` derives a conservative `low`/`medium`/`high` tier once per Canvas mount from `devicePixelRatio` and `navigator.hardwareConcurrency` only — no benchmarking, no telemetry, no user-agent sniffing. A typical current desktop/laptop resolves to `high`, which matches today's hardcoded Canvas config exactly (antialias on, ContactShadows on, DPR up to 1.35), so nothing changes for the common case; only clearly constrained devices (very high DPR or 1–2 logical cores) drop `ContactShadows`, disable antialias, and cap DPR at 1. The tier is decided once and never recomputed reactively, so it cannot itself trigger a Canvas remount.

Use the repeatable manual protocol in [device-benchmark.md](device-benchmark.md) for desktop, Android, and iOS/Safari measurements. The repository intentionally stores an empty results template rather than invented hardware numbers.
